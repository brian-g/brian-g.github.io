# WARP.md

This file provides guidance to WARP (warp.dev) when working with code in this repository.

## Overview

This is a static personal portfolio website (brian-g.github.io) built with HTML, LESS CSS, and jQuery. The site is hosted on GitHub Pages and showcases UX design work. The project uses CodeKit as its build tool rather than modern npm-based tooling.

## Build System

### CodeKit (Primary Build Tool)
This project uses **CodeKit** (https://codekitapp.com/) - a Mac-only GUI application for web development that handles:
- LESS compilation (`/less/*.less` → `/css/*.css`)
- JavaScript minification (`/js/*.js` → `/js/min/*-ck.js`)
- File watching and auto-compilation

**Important**: CodeKit is a GUI application, not a CLI tool. Build configuration is stored in:
- `codekit-config.json` - Project settings
- `config.codekit` / `config.codekit3` - Legacy config files

### Manual Build Process (Without CodeKit)
If CodeKit is not available, LESS files can be compiled using the `lessc` command:
```bash
# Install LESS compiler if needed
npm install -g less

# Compile main LESS file
lessc less/default.less css/default.css --compress
```

## Development Workflow

### Making Style Changes
1. Edit LESS files in `/less/` directory:
   - `default.less` - Main stylesheet (imports all others)
   - `constants.less` - Variables and configuration
   - `logo.less`, `symbols.less`, `animation.less` - Component styles
2. CodeKit will automatically compile to `/css/default.css`
3. If using manual compilation, run `lessc less/default.less css/default.css --compress`

### Making JavaScript Changes
1. Edit source files in `/js/` directory:
   - `default.js` - Main interactive functionality (lightbox, parallax)
   - `wurfl.js` - Device detection
2. CodeKit will automatically minify to `/js/min/default-ck.js`
3. Manual minification not typically needed for local development

### Testing Changes Locally
Since this is a static site, simply open `index.html` in a browser:
```bash
# Option 1: Open directly
open index.html

# Option 2: Use Python simple server
python3 -m http.server 8000
# Then navigate to http://localhost:8000

# Option 3: Use PHP server
php -S localhost:8000
```

### Deploying to GitHub Pages
```bash
# Commit changes
git add .
git commit -m "Description of changes"

# Push to GitHub (automatically deploys)
git push origin master
```

The site is automatically published to https://brian-g.github.io/ when changes are pushed to the master branch.

### Key Technical Details

**LESS Compilation Pipeline:**
- `less/default.less` imports all component files
- CodeKit compiles to `css/default.css` with compression
- Output style: compressed/minified
- Relative URLs: disabled
- Strict math/units: enabled

**JavaScript Architecture:**
- jQuery-based (loaded from CDN)
- `default.js` provides:
  - Lightbox modal for portfolio images
  - Image gallery navigation (next/previous)
  - Parallax scrolling effect on body
- CodeKit prepends dependencies and minifies to `/js/min/default-ck.js`

**No Build Dependencies:**
- No `package.json` or npm modules
- No webpack, vite, or other bundlers
- All dependencies loaded via CDN (jQuery, Modernizr)

### Styling System

**LESS Variables** (in `constants.less`):
- Typography: `@bodyFontFamily`, `@headingFontFamily`, `@bodyFontSize`
- Layout: `@grid`, `@gutter`, `@padding`
- Colors: `@accent0`, `@linkHover`, `@linkActive`
- Print styles: `@printGrid`, `@printBodyFontSize`, etc.

**Responsive Design:**
- Uses `@media` queries for print styles (portfolio can be printed as resume)
- Small device breakpoint: `@smallDeviceWidth`
- Minimal mobile-specific styling

### Content Architecture

The site is a **single-page portfolio** with sections:
1. **Header** - Name, contact links (LinkedIn, Twitter, Facebook, Medium)
2. **Intro** - Professional summary and design philosophy
3. **Resume** - Work experience at FarmQA and Microsoft
4. **Portfolio** - Project showcases with image galleries:
   - FarmQA Web App
   - FarmQA Scouting (mobile)
   - FarmQA Website
   - Microsoft Dynamics 365 Talent
   - Microsoft Dynamics AX
   - Web-based HRM
   - Project Time and Expense

**Lightbox Gallery System:**
- Click any portfolio thumbnail to open lightbox
- Navigate between images in same album
- Lightbox clones figure elements and shows fullres images
- Uses jQuery animations for smooth transitions

## Common Tasks

### Update Portfolio Content
Edit `index.html` sections directly - no template system or CMS.

### Add New Portfolio Item
1. Add images to `/Portfolio/` directory
2. Add corresponding HTML in the `#portfolio` section of `index.html`
3. Follow the `.portfolio-item.thumbnail-album` pattern for consistency

### Modify Site Colors/Typography
Edit LESS variables in `less/constants.less`, then recompile.

### Update SEO Meta Tags
Edit `<head>` section in `index.html` (title, description, og: tags).
